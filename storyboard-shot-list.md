## How to use this doc

Each scene below maps to one screen in the prototype's **Present mode**. Present mode is already sequenced in the same order as this shot list, so you can screen-record straight through with arrow keys, or rebuild frames natively from the highlights and motion notes.

- **Screen** — which mock/state to capture
- **Beat** — the story job this scene does
- **VO** — draft voiceover (also shown live in the prototype's caption bar)
- **On screen / highlight** — what the eye should land on
- **Motion** — suggested animation and how to trigger it in the mock

## The core device: background vs. what the recruiter sees

The single most important thing the film must communicate is **who is doing the work**. Two built-in tools make this explicit — use them in every scene:

- **Layer badge** (pinned, top-right of every screen): reads `⚙ AGENT · working in the background` (electric violet) or `👤 RECRUITER · what Robin sees` (amber). It updates automatically per screen. Keep it in frame — it's the viewer's constant compass.
- **Agent / Recruiter view toggle** (top bar, and in the present-mode stage bar): flips the whole prototype between two realities.
  - **Agent view** = the full detail of what the agent is doing (the default; use it for the "machine at work" act).
  - **Recruiter view** = Robin's reality. Background-agent screens get **veiled** with "The agent is handling this — Robin never sees it." Only the recruiter touchpoints (Mission Control, Step 3, Step 7) stay fully visible.

**The killer demonstration:** at any background step, toggle Agent → Recruiter on camera. The screen veils over. That single motion says "all of this happens without her." Recommended as the transition into Scene 9 (the human turn) and again at the close.

**Layer of each screen:** Mission Control = recruiter · Step 1,2,4,5,6,8,9 = agent (background) · Step 3,7 = recruiter · Flow overview = system.

## The narrative arc

The film has a simple three-act spine. Keep it in mind so individual scenes ladder up:

1. **The problem (Scene 1):** a recruiter buried in volume; no human can do it all.
2. **The machine (Scenes 2–9):** the agent runs the whole hiring loop autonomously — fast, montage energy.
3. **The payoff (Scenes 1 return / 11):** the recruiter is now strategic, doing only the human moments. Close back on Mission Control.

The emotional turn is *volume → leverage*: she didn't get more hours, she got an agent.

## Scene 1 — Cold open: Mission Control

- **Screen:** Mission Control (landing)
- **Beat:** Establish the protagonist and the scale of her job. Tension: 18 reqs, 412 candidates, impossible by hand.
- **VO:** "This is Robin. Eighteen open roles. Four hundred candidates in motion. No human can keep up with that — so she doesn't."
- **On screen / highlight:** KPI strip, especially the red **"7 Need your attention."** Then the ★ HERO STORY items in the queue.
- **Motion:** KPI numbers **count up from 0** (auto on load). Cards **rise/fade in** staggered. Hold on the red "7." Optional punch-in on the hero-tagged queue item.
- **Interactive beat (filmable):** Mission Control is now live — hover a queue card (a "Review →" hint appears), then click it to **open the context window**: the AI's reasoning, candidate/plan detail, and suggested next steps. Click the primary action and the item resolves — the **"Needs you" count drops 7 → 6** on screen. This "review context → act" moment is a strong thing to capture: it shows the recruiter deciding *with* the agent, not just rubber-stamping.
- **Cut tip:** Linger 3–4s — this is the "before." Let the volume feel almost overwhelming.

## Scene 2 — The loop revealed

- **Screen:** Flow overview (9 cards + "Role Filled")
- **Beat:** Reveal the system doing the work. This is the "how."
- **VO:** "Behind the scenes, an agent runs the hiring loop — nine steps, from a role opening to candidates sourced, screened and scheduled. (The offer stays human-led.) Let's follow one hire."
- **On screen / highlight:** All 9 step cards; end on card 1. Note that Steps 1–2 are bracketed as one **Prep phase** — the agent runs them back-to-back before the recruiter touches anything.
- **Motion:** Cards cascade in (stagger). Consider a quick camera push toward card 1 to transition into Scene 3.
- **Note:** Scenes 3 and 4 (Understand the Role + Build the Talent Plan) are the Prep phase. Play them as one continuous agent beat, then land on the plan the recruiter reviews in Scene 5.

## Scene 3 — Understand the Role

- **Screen:** Step 1
- **Beat:** The agent is smart from second one — it reads the job and predicts outcomes.
- **VO:** "The moment the role opens, it reads the job, learns from past hires, and predicts how hard this will be to fill."
- **On screen / highlight:** The **confidence ring (82)** and **time-to-fill (38 days)**; then the skills chips.
- **Motion:** Ring **fills 0→82** and number **counts up** (auto). Skill chips can pop in sequentially if rebuilding.

## Scene 4 — Build the Talent Plan

- **Screen:** Step 2
- **Beat:** It doesn't just search one place — five talent pools, prioritized.
- **VO:** "It builds a plan across five talent pools — top matches, silver-medalists, internal talent, referral champions, and outside only if it has to."
- **On screen / highlight:** The four pool count cards, then the prioritized execution table (row 1 = Marcus Lee).
- **Motion:** Pool cards rise in; count numbers up. Table rows wipe in top-to-bottom. Spotlight the Marcus Lee row (hero continuity).

## Scene 5 — Recruiter Review (the human's first touch)

- **Screen:** Step 3
- **Beat:** Robin stays in control — but effortlessly. One plan, one click.
- **VO:** "Robin sees one plan, not a hundred tasks. Approve — and nothing goes out until she says so."
- **On screen / highlight:** The consolidated checklist, then the **Approve All** button.
- **Motion:** **Click Approve All** live — button morphs to "✓ Approved — executing" and turns green. This is a satisfying beat; hold on it.

## Scene 6 — Execution (montage energy)

- **Screen:** Step 4
- **Beat:** The machine goes to work — everywhere at once.
- **VO:** "Now it executes — four tracks in parallel. Personalized outreach, profile refreshes, referral campaigns, internal notices."
- **On screen / highlight:** All four track cards; the live sample outreach message at the bottom.
- **Motion:** Four cards animate in near-simultaneously (parallelism = speed). Checklist items tick in. Consider fastest pacing of the film here.

## Scene 7 — Qualify & Score

- **Screen:** Step 5
- **Beat:** It reads every reply and sorts signal from noise.
- **VO:** "As replies come in, it qualifies every one — interested, qualified, or needs a human."
- **On screen / highlight:** The four status count cards, then the evaluation table. Highlight Marcus = **Qualified**.
- **Motion:** Status counts up; table rows wipe in. Color-code the status tags as they land.

## Scene 8 — Orchestrate & Schedule

- **Screen:** Step 6
- **Beat:** A dedicated **scheduling agent** carries candidates forward and eats the scheduling pain — up to the interview. Offer & close stay human-led (out of the agent's scope, on purpose).
- **VO:** "Then a scheduling agent takes over — coordinating calendars, panels, time zones and reminders, right up to the interview. The offer stays human-led."
- **On screen / highlight:** The **progression timeline** (Marcus advancing; note the final step hands to the hiring team) and the scheduling-agent checklist.
- **Motion:** Timeline dots advance one by one; the "now" dot pulses (amber). Good spot for a subtle progress-line draw-on.

## Scene 9 — Human Escalation (the turn)

- **Screen:** Step 7
- **Beat:** The most important idea in the film — the agent knows its limits and hands the human moments to the human.
- **VO:** "And it knows when to stop. Competing offers. Executives. A quiet candidate. It says: I recommend you call this one."
- **On screen / highlight:** The escalation cards; land hard on the **"I recommend calling this candidate"** callout.
- **Motion:** Escalation rows **slide in** from the right. Hold on the recommendation banner — this is the thesis beat, give it air.

## Scene 10 — Continuous Optimization

- **Screen:** Step 8
- **Beat:** It's always watching and self-correcting.
- **VO:** "It never sleeps — watching pipeline health and fixing problems before they cost you the hire."
- **On screen / highlight:** Health signal bars; the auto-recommended actions list; the amber "Pipeline weakening" tag.
- **Motion:** Bars **fill left-to-right**. Recommendation list ticks in.

## Scene 11 — Role Filled (payoff)

- **Screen:** Step 9
- **Beat:** Win. Quantify it and close the loop back to learning.
- **VO:** "Role filled — seven days ahead of prediction. And every hire teaches the agent to do the next one better."
- **On screen / highlight:** The outcome metrics (31 days, referral win, 94% accuracy); the "feeds back to Step 1" note.
- **Motion:** Metrics count up; a celebratory but restrained accent (no confetti overload). Draw a subtle arrow back toward Step 1 to signal the loop.

## Scene 12 — Close: back to Mission Control

- **Screen:** Mission Control (return)
- **Beat:** Payoff for the human. Same screen as the open, but now it *reads as calm control*, not chaos.
- **VO:** "So Robin spends her day on the seven moments that need a person — not the four hundred that don't. That's Mission Control."
- **On screen / highlight:** Same KPI strip; end on the agent-loop panel showing AUTO vs YOU. Optionally trigger the **7 → 6** count drop as one item is cleared.
- **Motion:** Reuse the open framing for bookend symmetry. Final logo/endcard.

## Production notes for the designer

**Capturing from the prototype**
- Open `ai-recruiting-agent-flow.html`, click **▶ Present** (top right). Sidebar/chrome hide, type scales up, a caption bar appears with the VO line.
- **→ / Space** = next scene, **←** = previous, **Esc** = exit. Scenes are pre-ordered to match this doc.
- Record at **1920×1080**. The layout is responsive; maximize the window before recording.

**Legibility for video**
- Present mode already bumps type size, but tables (Steps 2, 5) are still dense. For final frames, consider isolating 3–4 rows or rebuilding them larger. One idea per frame reads on video; a full table does not.
- Avoid capturing the browser cursor over text. Hide the pointer between clicks.

**Continuity rules (don't break the story)**
- Always keep the hero thread visible: **ENG-4471 / Marcus Lee / Priya Nair**. Wherever a list appears, the hero row is spotlighted (★ HERO).
- Purple = the agent acting. Amber = a human moment. Keep that color logic consistent in any custom motion graphics.

**Style spec — Eightfold brand palette (pulled from eightfold.ai)**
- Ink-navy (primary) `#191841`, deep `#10102E`, tints `#EEF0F7` / `#D8DCEC`
- Signature mint (accent, used in gradients/bars) `#88E2D2`
- Electric violet (AI / agent) `#5E00FF`
- Slate body text `#484B58`, lines `#E4E7F0`, app background `#F7F8FC`
- Status: green `#12A150`, amber `#C77700`, red `#D3263B`, blue `#2563EB`
- Signature gradient: navy → mint (`#191841` → `#88E2D2`) for logo, bars, rings, hero accents
- Color logic: **navy** = brand/primary · **violet** = AI/agent · **mint** = accent/highlight · **amber** = human moment
- Type: system UI stack (Inter / SF equivalent). Card radius 14px. Soft shadow, low spread.

**Suggested music/pacing**
- Act 1 (Scene 1): sparse, slightly tense.
- Act 2 (Scenes 2–8): build tempo; Scene 6 (Execution) is the peak of momentum.
- Scene 9 (Escalation): pull back — this is the human beat, let it breathe.
- Act 3 (11–12): resolve, warm, confident.
